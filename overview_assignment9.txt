Log Monitoring System using EventEmitter

Overview:
A Node.js project that simulates a real-time log monitoring system using Event Emitters.

**logger.js: Generates random log messages.

**monitor.js: Listens for critical logs (ERROR, HIGH_CPU, DISK_FULL), writes them to alerts.log, and shows a summary every 5 alerts.

**alerts.log – Stores critical logs.

=>Logs are generated every 5 seconds.
=>Critical logs are saved to alerts.log.
=>Summary is shown every 5 critical alerts.

2. Explain difference ways in which child process can be created and difference b/w them.

Child Process Demonstration using Node.js

Overview:
This project explores the four different ways to create child processes in Node.js using the child_process module.

Concepts Covered:
    spawn – Best for large stream-based output.
    exec – Buffers output, good for small results.
    execFile – Runs files directly, avoids shell injection.
    fork – Specialized for Node-to-Node communication with IPC.
